Lab 3: Dynamic Memory - Learning From Conversation¶

In this lab, you will complete the memory system by enabling dynamic profile extraction.

You will activate the profile extraction mechanism that automatically learns user preferences from natural conversation and updates their profile in real-time, without requiring explicit commands or forms.

By the end of this lab, you will:

  • ✅ Enable automatic preference extraction from conversation
  • ✅ Build user profiles organically through natural dialogue
  • ✅ Update profiles incrementally as new information emerges
  • ✅ Understand how AI context providers enable bidirectional memory

Step 1: Understand the Extraction Mechanism¶

The UserProfileMemoryProvider has two methods: - ProvideAIContextAsync: Loads profile (✅ Already working in Lab 2) - StoreAIContextAsync: Extracts and saves preferences (🔄 We'll activate this now)

Open src/backend/Services/UserProfileMemoryProvider.cs:

Review the StoreAIContextAsync method:

public async Task StoreAIContextAsync(
    AIContext context, 
    CancellationToken cancellationToken)
{
    // 1. Get conversation history
    var messages = context.MemoryContext.Messages;

    // 2. Send to LLM with extraction instructions
    var extractionPrompt = """
        Analyze the conversation and extract travel preferences.
        Return ONLY new/updated information as JSON.
        """;

    // 3. Parse extracted JSON
    var extractedProfile = JsonSerializer.Deserialize<UserProfileMemory>(response);

    // 4. Merge with existing profile
    var currentProfile = await LoadProfileFromCosmosAsync();
    var mergedProfile = MergeProfiles(currentProfile, extractedProfile);

    // 5. Save back to Cosmos DB
    await SaveProfileToCosmosAsync(mergedProfile);
}

How it works: 1. After agent responds, StoreAIContextAsync is called automatically 2. Recent conversation history sent to LLM with extraction prompt 3. LLM identifies preferences: budget, interests, dietary needs, travel style 4. Extracted data merged with existing profile (incremental updates) 5. Updated profile saved to Cosmos DB 6. Next conversation loads updated profile via ProvideAIContextAsync

No code changes needed - the functionality is already there!

Step 3: Understand the Extraction Prompt¶

Open src/backend/Services/UserProfileMemoryProvider.cs and find the extraction prompt:

private const string ExtractionInstructions = """
You are a profile extraction assistant. Analyze the conversation and extract 
travel preferences into structured JSON format.

ONLY extract information that is EXPLICITLY mentioned or clearly implied.
Return ONLY fields that have new or updated information.

Profile Schema:
{
  "travelStyle": "adventure seeker | luxury | budget backpacker | family | relaxation | cultural",
  "budgetRange": "$1000-2000 | $2000-3000 | $3000-5000 | $5000+",
  "interests": ["hiking", "beaches", "museums", "food", "wildlife"],
  "dietaryRequirements": "vegetarian | vegan | gluten-free | halal | kosher | none",
  "numberOfTravelers": 1,
  "tripDuration": "weekend | 1 week | 2 weeks | 1 month+",
  "pastDestinations": [
    {
      "destination": "Place name",
      "rating": "loved it | okay | disappointing"
    }
  ]
}

Rules:
- Keep interests list to top 5 most relevant
- Infer travel style from context (e.g., "stressed at work" → "relaxation")
- Don't hallucinate - only extract what's actually mentioned
- Merge new info with existing profile (don't remove old data)

Return ONLY the JSON object with updated fields.
""";

Key Points: - LLM acts as intelligent parser - Extracts structured data from unstructured conversation - Incremental updates (doesn't overwrite entire profile) - Handles implicit preferences (e.g., "stressed at work" implies relaxation)

Profile Merging Strategy¶

The MergeProfiles method combines old and new data intelligently:

private UserProfileMemory MergeProfiles(
    UserProfileMemory existing, 
    UserProfileMemory extracted)
{
    return new UserProfileMemory
    {
        // Use new value if provided, otherwise keep existing
        TravelStyle = extracted.TravelStyle ?? existing.TravelStyle,
        BudgetRange = extracted.BudgetRange ?? existing.BudgetRange,
        DietaryRequirements = extracted.DietaryRequirements ?? existing.DietaryRequirements,

        // Merge lists (combine + deduplicate)
        Interests = MergeLists(existing.Interests, extracted.Interests),
        PastDestinations = MergeLists(existing.PastDestinations, extracted.PastDestinations),

        // Numeric fields: use new if provided
        NumberOfTravelers = extracted.NumberOfTravelers ?? existing.NumberOfTravelers,
    };
}

Merge Behavior: - Strings: New overwrites old (if provided) - Lists: Combine and deduplicate (keeps unique items) - Numbers: New overwrites old (if provided) - Nulls: Ignored (doesn't erase existing data)

When Extraction Runs¶

StoreAIContextAsync is called in these scenarios: 1. After agent provides final response 2. After tool calls complete (if any) 3. Before returning response to user 4. Only if conversation history exists (skip on empty conversations)

Performance: Extraction adds ~1-2 seconds to response time (LLM call to parse preferences)

  • Start conversation with existing profile
  • Mention NEW preferences in conversation
  • Verify profile updated (check Cosmos DB or new conversation)
  • Confirm old preferences not lost

Test 3: Implicit Extraction¶

  • Say "I've been stressed at work" → should infer relaxation style
  • Say "traveling with my family" → should set numberOfTravelers
  • Say "I loved my trip to Tokyo" → should add to pastDestinations

Test 4: Cross-Session Persistence¶

  • Build profile in conversation
  • Close browser completely
  • Reopen and start new conversation
  • Verify profile still loads correctly

Observations¶

After implementing dynamic memory, you should notice:

  • ✅ Profile builds automatically from conversation
  • ✅ No forms or explicit "save profile" commands needed
  • ✅ Preferences update incrementally as user mentions new information
  • ✅ Agent becomes more personalized over time
  • ✅ Natural conversation flow (not interrogation)

Architecture Benefits¶

This pattern provides several advantages:

  1. User Experience: Natural conversation vs filling out forms
  2. Data Quality: Preferences emerge organically, more accurate
  3. Maintenance: No manual profile management
  4. Privacy: Users control what they share through conversation
  5. Flexibility: Works for implicit and explicit preference statements
