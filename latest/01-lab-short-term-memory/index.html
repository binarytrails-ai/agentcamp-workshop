Lab 1: Short-Term Memory - Maintaining Context¶

Welcome to your first hands-on lab! In this lab, you'll learn about short-term memory. This is required for any agent to maintain context within a conversation to respond accurately.

By the end of this lab, you will:

  • ✅ Understand short-term memory
  • ✅ Test how agents maintain context across messages

Understanding Short-Term Memory¶

Short-term memory in AI agents is the conversation context that persists within a single session. It's how the agent remembers what was said earlier in the conversation.

When you chat with an AI agent, each message you send includes:

  1. The conversation history - Previous messages in the session
  2. Your new message - The current input
  3. System instructions - The agent's guidelines

The language model processes all of this together to generate a contextually relevant response.

[System]: You are a travel assistant...
[User]: I want to plan a trip
[Assistant]: I'd be happy to help! Where would you like to go?
[User]: Somewhere with beaches    <-- This message includes all the above
[Assistant]: Great choice! [generates response based on full context]

Context Window Limitations¶

Language models have a context window which limits how much conversation history they can consider at once. For example:

  • GPT-4o: 128,000 tokens
  • GPT-4o-mini: 128,000 tokens

When conversations get very long, older messages may need to be removed or summarized to stay within the limit.

Instructions¶

Step 1: Understand the Agent Configuration¶

Open src/backend/Agents/ContosoTravelAgentBuilder.cs and review how the agent is configured using the Microsoft Agents Framework.

 AIAgent agent = _chatClient.AsAIAgent(new ChatClientAgentOptions
        {
            Name = Constants.AgentName,
            ChatOptions = new()
            {
                Instructions = AgentInstructions,
                Tools = []
            },

        });
  • Name: Name of the agent for logging and telemetry
  • ChatOptions.Instructions: The system prompt that defines the agent's behavior
  • ChatOptions.Tools: List of tools the agent can call

While this lab uses a minimal configuration, ChatOptions supports many additional settings - Temperature, MaxOutputTokens, TopP, ResponseFormat and more to control the agent's responses.

The agent is built on top of Microsoft.Extensions.AI.IChatClient, which provides a unified interface for different LLM providers. This means you can switch between models (Azure OpenAI, OpenAI, etc.) without changing your agent code.

Test Your Implementation¶

Refer to the Running the Application Locally section in the Environment Setup guide to start the application.

Try the following conversation with the agent:

User: Can you help me plan a trip?

The assistant should ask you for more details. Then continue the conversation by answering with more information about your travel preferences (e.g., "adventure travel", "family-friendly", "beaches", etc.) and see how the agent maintains context across messages to provide relevant responses.

From the Trace window in the .NET Aspire dashboard, you can see the conversation history between the user and the agent.
