Lab 2: Static Profile Memory - Personalized Recommendations¶

In this lab, you will enhance your travel assistant by adding static user profile memory.

You will integrate a pre-configured user profile that gets automatically injected into each conversation, enabling personalized destination recommendations without requiring users to repeat their preferences.

By the end of this lab, you will:

  • ✅ Understand how AI context providers inject data before LLM calls
  • ✅ Enable the agent to use pre-loaded user profiles for personalization
  • ✅ Provide personalized recommendations based on stored preferences
  • ✅ See the benefits of persistent user context

The Problem: Repeating Yourself Every Time¶

From Lab 1, you saw that without memory, users must specify preferences repeatedly:

👤 User: I want to plan a trip

🤖 Agent: I can help! What are your interests?

👤 User: I like hiking and local cuisine

🤖 Agent: Great! What's your budget?

👤 User: Around $2,000-$3,000

🤖 Agent: Perfect! Here are some destinations...

[Next day, new conversation]

👤 User: Suggest a vacation for me

🤖 Agent: What are your interests? ❌ Asking again!

Sample Interaction: What You'll Build¶

With static profile memory, the agent remembers your preferences:

👤 User: I want to plan a trip

🤖 Agent: Hi! I see you're an adventure seeker who loves hiking, local cuisine, 
          and photography. Your usual budget is $2,000-$3,000, and you prefer 
          vegetarian dining options.

          Would you like me to suggest destinations based on these preferences, 
          or are you looking for something different this time?

👤 User: Yes, use those preferences

🤖 Agent: Perfect! Based on your love for hiking, local cuisine, and photography, 
          here are some great destinations within your $2,000-$3,000 budget:

• Blue Mountains, New South Wales
  - Dramatic cliffs and extensive hiking trails
  - Excellent farm-to-table dining scene with vegetarian options

  - Budget: $2,500 for 4 days including accommodation and activities

• Grampians National Park, Victoria
  - Rugged mountain hiking with Aboriginal rock art
  - Local wineries and artisan food producers (great veggie options)
  - Photography: Waterfalls, wildlife, mountain vistas
  - Budget: $2,200 for 4 days

• Abel Tasman National Park, New Zealand
  - Coastal hiking trails with turquoise waters
  - Fresh seafood and local produce (excellent vegetarian cafes)
  - Photography: Golden beaches, native forests, kayaking scenes
  - Budget: $2,800 for 5 days

Which of these appeals to you?

[Next day, new conversation]

👤 User: Suggest a vacation

🤖 Agent: Hi again! Based on your adventure seeker profile, love for hiking 
          and photography, and your vegetarian preferences - would you like 
          more suggestions similar to what we discussed before, or something new?

Key Improvement: Agent remembers your profile across conversations, saving time and providing better recommendations immediately.

Open src/backend/Services/UserProfileMemoryProvider.cs:

This class implements IAIContextProvider which has two key methods:

public class UserProfileMemoryProvider : IAIContextProvider
{
    // Called BEFORE sending messages to the LLM
    // Loads user profile and injects it into the conversation context
    public async Task ProvideAIContextAsync(
        AIContext context, 
        CancellationToken cancellationToken)
    {
        // 1. Load profile from Cosmos DB
        // 2. Format profile as context message
        // 3. Inject into conversation BEFORE agent sees it
    }

    // Called AFTER receiving LLM response  
    // Extracts preferences from conversation and updates profile
    public async Task StoreAIContextAsync(
        AIContext context, 
        CancellationToken cancellationToken)
    {
        // Used in Lab 3 - not active yet
    }
}

How it works: 1. Agent receives user message 2. ProvideAIContextAsync loads profile from database 3. Profile injected as system context: "TRAVELER PROFILE: adventure seeker, budget $2000-3000..." 4. LLM sees user message + profile context 5. LLM generates personalized response

Add the AIContextProviderFactory to enable profile injection:

public Task<AIAgent> CreateAsync()
{
    var agent = _chatClient.CreateAIAgent(new ChatClientAgentOptions
    {
        Name = Constants.AgentName,
        ChatOptions = new()
        {
            ResponseFormat = ChatResponseFormat.Text,
            Instructions = AgentInstructions,
            Tools = [
                AIFunctionFactory.Create(
                    (string keywords) => DestinationSearchTool.SearchDestinations(
                        keywords, _cosmosDatabase, _config.CosmosDbDestinationsContainer, _logger),
                    name: "SearchDestinations",
                    description: "Search for travel destinations based on keywords...")
            ]
        },
        AIContextProviderFactory = (ctx) =>
        {
            // Extract userId from HTTP context
            string userId = _httpContextAccessor.HttpContext?.Items["UserId"] as string 
                            ?? "demo-user"; // Default to demo-user for testing

            // Create memory provider scoped to this user
            var userProfileMemoryProvider = new UserProfileMemoryProvider(
                _chatClient,
                _cosmosDatabase,
                _config.CosmosDbUserProfileContainer,
                new UserProfileMemoryProviderScope
                {
                    UserId = userId,
                    ApplicationId = Constants.ApplicationId,
                    AgentId = Constants.AgentName
                },
                _loggerFactory.CreateLogger<UserProfileMemoryProvider>()
            );

            return userProfileMemoryProvider;
        }
    });

    agent.AsBuilder()
        .UseOpenTelemetry(Constants.ApplicationId, options => options.EnableSensitiveData = true)
        .UseLogging(_loggerFactory)
        .Build();

    return Task.FromResult(agent);
}

What changed: - Added AIContextProviderFactory callback - Extracts userId from HTTP context (defaults to "demo-user") - Creates UserProfileMemoryProvider scoped to user/application/agent - Provider automatically loads profile before each LLM call

- Acknowledge their preferences naturally at the start of the conversation

 Would you like suggestions based on these preferences?"

If they confirm, proceed with personalized recommendations WITHOUT asking again.

When NO profile exists:

Use the SearchDestinations tool to find relevant destinations based on:
- User interests (from profile or conversation)

When presenting destinations:

- Highlight experiences that align with their interests
- Note vegetarian/dietary-friendly options if relevant

- Be conversational and enthusiastic
- Reference profile details naturally (not mechanical listing)
- Provide specific, actionable recommendations

Key Implementation Details¶

How Profile Injection Works¶

  1. Request Flow: 

    User Message 
→ UserProfileMemoryProvider.ProvideAIContextAsync()
→ Load profile from Cosmos DB
→ Format as context message
→ Inject into conversation
→ LLM processes message + profile context
→ Generate personalized response

  2. Profile Context Format: 

    === TRAVELER PROFILE ===
Travel Style: adventure seeker
Budget Range: $2,000-$3,000
Interests: hiking, local cuisine, photography
Dietary Requirements: vegetarian
Number of Travelers: 1

  3. Scoping:

  4. Profiles are scoped by userId, applicationId, and agentId
  5. Document ID: {userId}_{applicationId}_{agentId}
  6. Ensures isolation between users and agents

Static vs Dynamic Memory¶

Lab 2 (Current): Static Memory - Profile pre-loaded from database - Used for context injection - NOT updated during conversation - StoreAIContextAsync not yet active

Lab 3 (Next): Dynamic Memory - Profile built FROM conversation - Automatically extracted and updated - StoreAIContextAsync enabled - Learns from natural dialogue
