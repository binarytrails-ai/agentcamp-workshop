Lab 1: Baseline Search - Finding Destinations Without Memory¶

In this lab, you will implement a basic travel destination search feature using keyword-based querying.

You will create a simple search tool that queries a Cosmos DB database for travel destinations based on user interests, without any personalization or memory capabilities.

By the end of this lab, you will:

  • ✅ Implement a Cosmos DB search tool for finding destinations
  • ✅ Integrate the search tool with your AI agent
  • ✅ Understand baseline agent behavior without memory
  • ✅ See the limitations of stateless, non-personalized recommendations

The Problem: Every Search Starts From Scratch¶

Without memory or personalization, users must specify their preferences every single time:

👤 User: I want to go somewhere with beaches

🤖 Agent: I can help you find beach destinations! Let me search...

Based on your interest in beaches, here are some great options:

• Bondi Beach, Sydney
  - Iconic surf beach with vibrant coastal culture
  - Category: Beach & Coastal
  - Best time: October-April

• Great Barrier Reef, Queensland  
  - World's largest coral reef system with tropical islands
  - Category: Beach & Coastal, Nature & Wildlife
  - Best time: June-October

Would you like more information about any of these?

[New conversation]

👤 User: Suggest a beach vacation

🤖 Agent: I can help! Let me search for beach destinations...

[Same generic results - no personalization, no memory of previous searches]

Key Implementation Details¶

How the Search Works¶

  1. User Input: User provides keywords or interests (e.g., "beaches", "hiking", "adventure")

  2. Tool Invocation: Agent calls SearchDestinations tool with keywords

  3. Database Query: Tool searches Cosmos DB using CONTAINS clauses: 

    WHERE CONTAINS(c.category, 'beach') 
   OR ARRAY_CONTAINS(c.experiences, 'beach')
   OR CONTAINS(c.description, 'beach')

  4. Result Formatting: Returns structured data with destination name, category, description, experiences

  5. Agent Response: Agent formats results conversationally and presents to user

Limitations of This Approach¶

  • No Personalization: Same results for all users with similar keywords
  • No Memory: Each conversation starts fresh
  • No Context: Can't combine preferences across multiple turns
  • No Learning: Doesn't improve recommendations based on user feedback

These limitations will be addressed in Labs 2 and 3!

Observations¶

After testing, you should notice:

  • ✅ Search works well for single, clear keywords
  • ✅ Results are relevant but generic
  • ❌ Agent doesn't remember preferences within a conversation
  • ❌ Agent doesn't combine multiple interests intelligently
  • ❌ No personalization based on budget, travel style, or past trips
